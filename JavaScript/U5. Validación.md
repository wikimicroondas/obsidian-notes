## Validación JSON: Schema
Conjunto de protocolos para asegurar identidades inequívocas a objetos y controlar el acceso de tipos de datos


"$schema"
"$id"
"title"
"description"
"properties"


urls: uri
números: minimum, maximum, multipleOf

### Asociación con documentos JSON
Podemos validar los datos mediante plantillas ya hechas `tipo.schema.json`.

```json
{
  "json.schemas": [
    {
      "fileMatch": ["datos/usuarios.json"],
      "url": "./schemas/usuario.schema.json"
    },
    {
      "fileMatch": ["datos/productos.json"],
      "url": "./schemas/producto.schema.json"
    },
    {
      "fileMatch": ["datos/pedidos.json"],
      "url": "./schemas/pedido.schema.json"
    }
  ]
}
```
## Intercambio de información con APIs
### Acciones HTTP

| Acción    | Significado |
| --------- | ----------- |
| GET       | Leer        |
| POST      | Crear       |
| Delete    | Eliminar    |
| Put/Patch | Actualizar  |
CRUD
> Create, Read, Update, Delete

### Códigos

| Código | Significado           | Descripción                             |
| ------ | --------------------- | --------------------------------------- |
| 200    | OK                    | La petición fue exitosa                 |
| 201    | Created               | Recurso creado exitosamente (POST)      |
| 204    | No Content            | Éxito, sin datos que devolver (DELETE)  |
| 400    | Bad Request           | Petición mal formada o datos inválidos  |
| 401    | Unathorized           | Falta autenticación (API key, token)    |
| 403    | Forbidden             | Sin permiso para acceder al recurso     |
| 404    | Not Found             | El recurso solicitado no existe         |
| 429    | Too Many Requests     | Excediste el límite de peticiones       |
| 500    | Internal Server Error | Error en el servidor (no es culpa tuya) |
| 503    | Service Unavailable   | Servidor temporalmente caído            |
- **2xx** — Éxito
- **4xx** — Error del cliente (tú hiciste algo mal)
- **5xx** — Error del servidor

### Async/Await

Forma moderna de implementar las promesas, evita congelar procesos

```js
async function obtenerUsuarios() {
  try {
    const response = await
    fetch('https://jsonplaceholder.typicode.com/users');
    
    if (!response.ok) {
      throw new Error(`Error HTTP: ${response.status}`);
    }
    
    const usuarios = await response.json();
    console.log(usuarios);
    return usuarios;
  } catch (error) {
    console.error('Error al obtener usuarios:', error);
  }
}
```
