# Análisis DAFO

Es una herramienta de planificación estratégica fundamental que se utiliza para evaluar la situación interna y externa de una empresa, proyecto o persona. 

| **Interno** (Controlable) | **Externo** (Previsible) |
| :-----------------------: | :----------------------: |
|        Debilidades        |         Amenazas         |
|        Fortalezas         |      Oportunidades       |
Permite identificar ventajas competitivas, detectar puntos débiles, anticipar riesgos y apoyar la toma de decisiones informadas.

## Análisis CAME
El **Análisis CAME** es el que define las acciones concretas basándose en ese diagnóstico

| Corregir las Debilidades   |
| -------------------------- |
| Afrontar las Amenazas      |
| Mantener las Fortalezas    |
| Explotar las Oportunidades |
## Modelos a explorar
Para identificar factores externos podemos basarnos en los siguientes modelos:
- **PESTEL:** Para el macroentorno (Políticos, Económicos, Sociales, Tecnológicos, Ecológicos y Legales).
- **5 Fuerzas de Porter:** Para el microentorno (competencia, proveedores, clientes, productos sustitutivos y nuevos entrantes).

# Diagrama de Gantt

El Diagrama de Gantt es una herramienta visual esencial para la gestión de proyectos, ya que muestra el calendario de tareas y su progreso a lo largo del tiempo.

## ¿Para qué sirve el Diagrama de Gantt?

- Planificación temporal: define las fechas de inicio y fin de cada actividad mediante un eje horizontal de tiempo.
- Visualización de dependencias: permite ver qué tareas deben finalizar antes de que otras puedan comenzar.
- Seguimiento del progreso: facilita la identificación de retrasos y el estado general del proyecto de un solo vistazo.
- Hitos: marca eventos críticos o entregas clave del proyecto, representados habitualmente mediante un diamante.

# Diagrama de Casos de Uso

Un diagrama de casos de uso es una herramienta de UML (Lenguaje de Modelado Unificado) que describe cómo un usuario interactúa con un sistema para alcanzar un objetivo concreto. Se utiliza para definir de forma clara las funcionalidades del sistema desde el punto de vista del usuario.

## Elementos principales

- Actor: representa un rol externo que interactúa con el sistema, como una persona, otro software o un dispositivo.
- Caso de uso: se representa mediante una elipse y describe una función completa que realiza el sistema (por ejemplo, “Iniciar sesión” o “Realizar compra”).
- Sistema: se muestra como un rectángulo que delimita qué funcionalidades están incluidas en el desarrollo y cuáles quedan fuera.

## Relaciones clave

- Asociación: línea simple que conecta a un actor con un caso de uso.
- Include (Incluye): indica que un caso de uso es obligatorio para que otro pueda completarse.
- Extend (Extiende): representa un comportamiento opcional o excepcional que se ejecuta bajo determinadas condiciones.
- Generalización: muestra una relación de herencia, donde un actor o caso de uso específico hereda características de uno más general.
